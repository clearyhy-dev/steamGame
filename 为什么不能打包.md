# 为什么不能打包？—— 原因说明

## 为什么之前可以打包，现在不行了？

项目代码没变，**变的是你本机的环境或使用方式**，通常只有下面几类：

| 变化 | 说明 |
|------|------|
| **1. 用了国内镜像** | 系统或 Cursor 里把 `PUB_HOSTED_URL` 设成了 `https://pub.flutter-io.cn`，该镜像有时 TLS 出错，`pub get` 失败 → 打包失败。之前能打包可能是当时没用这个镜像，或镜像当时正常。 |
| **2. 在 Cursor 里打包** | Cursor 会起多个 dart 进程占着 Flutter 的“启动锁”，在 Cursor 终端里执行 `flutter build apk` 会一直卡在 “Waiting for... startup lock”。之前若能打包，多半是在**外面** cmd 打的。 |
| **3. JAVA_HOME 变了** | 换了 JDK 或路径带空格，Gradle 找不到 Java 会报错。已建议用 `E:\java\jdk-11.0.9` 并在打包脚本里写死。 |

**结论**：不是“这个版本不能打包”，而是**环境/方式**导致；按下面「解决方案」做即可恢复。

---

## 解决方案（按顺序做）

### 第一步：用脚本一键打包（推荐）

1. **关掉 Cursor**（避免抢锁）。
2. 打开 **资源管理器**，进入 **e:\steamGame**。
3. **双击运行** **`build_apk_full.bat`**。
4. 脚本会自动：结束 dart 进程、删锁、设官方 pub 源、设 JAVA_HOME、执行 `pub get` 和 `flutter build apk --release`。
5. 成功时 APK 在：`e:\steamGame\build\app\outputs\flutter-apk\app-release.apk`。

若脚本报错，看窗口里**第一段红色/错误信息**，对照下面「第二步」排查。

### 第二步：脚本仍失败时逐项检查

- **报错里带 “pub get failed” / “TLS” / “pub.flutter-io.cn”**  
  → 脚本已用官方源，若仍报错，多半是**网络**（代理、防火墙）。可尝试：  
  - 手机开热点给电脑用再运行一次脚本；或  
  - 临时把系统环境变量 `PUB_HOSTED_URL` 和 `FLUTTER_STORAGE_BASE_URL` 删掉，再运行脚本。

- **报错里带 “JAVA_HOME” / “java” / “JDK”**  
  → 确认本机存在目录 **E:\java\jdk-11.0.9**（和脚本里一致）。若 JDK 装在其他路径，用记事本打开 `build_apk_full.bat`，把 `set JAVA_HOME=E:\java\jdk-11.0.9` 改成你的 JDK 路径后保存，再运行。

- **一直卡在 “Waiting for... startup lock”**  
  → 说明脚本里的 `taskkill dart` 没生效（例如 Cursor 又起了 dart）。请**完全退出 Cursor** 后再双击运行 `build_apk_full.bat`。

### 第三步：不想用脚本时的手动打包

在**系统自带的 cmd**（不要用 Cursor 终端）里执行：

```bat
set PUB_HOSTED_URL=https://pub.dev
set FLUTTER_STORAGE_BASE_URL=https://storage.googleapis.com
set JAVA_HOME=E:\java\jdk-11.0.9
cd /d e:\steamGame
flutter pub get
flutter build apk --release
```

## 打包 Google Play 用 AAB（target API 35，需 JDK 17）

Google Play 要求应用 **targetSdkVersion 35**。项目已升级为 **compileSdk 35 / targetSdk 35**，使用的 Android Gradle Plugin 8.6 需要 **JDK 17**（你当前的 JDK 11 只能打 APK / 旧版 AAB，不能用于此构建）。

- **打 AAB 上传 Play**：请安装 **JDK 17**（如 [Adoptium](https://adoptium.net/)），将 JDK 17 放在 `E:\java\jdk-17` 或设置 `JAVA_HOME` 指向 JDK 17，然后双击运行 **`打包GooglePlay_AAB.bat`**。生成的 AAB 在 `build\app\outputs\bundle\release\app-release.aab`。
- **仅打 APK 装手机**：可继续用 JDK 11，但需把 `android/app/build.gradle` 里 `compileSdkVersion` / `targetSdkVersion` 暂时改回 28，并用原来的 `build_apk_full.bat` 等脚本打包（该 APK 不能用于上传 Play）。

## 一、Flutter 启动锁被占用（最常见）

### 现象
执行 `flutter build apk` 时一直卡在：
```text
Waiting for another flutter command to release the startup lock...
```

### 原因
- Flutter 同一时间只允许一个命令使用“启动锁”（锁文件在 SDK 的 `bin/cache/lockfile`）。
- **Cursor / VS Code 里** 的 Dart/Flutter 插件会常驻多个 **dart.exe** 进程（分析、补全等），它们会占用这个锁。
- 你在 Cursor 终端里再跑 `flutter build apk`，就会一直等这个锁，看起来像“卡住、不能打包”。

### 解决办法（任选一种）

1. **在 Cursor 外面打包（推荐）**  
   - 关掉 Cursor（或至少关掉所有 Flutter 运行/调试）。  
   - 打开系统自带的 **cmd** 或 **PowerShell**，执行：  
     ```bat
     cd /d e:\steamGame
     flutter build apk --release
     ```  
   - 或直接双击项目里的 **`build_apk_full.bat`**。  
   这样没有 IDE 的 dart 进程抢锁，一般就能正常打完包。

2. **不关 Cursor，先释放锁再打包**  
   - 在任务管理器里结束所有 **dart.exe**（会暂时影响 Cursor 的 Dart 补全等，过几秒会恢复）。  
   - 然后立刻在 Cursor 终端执行：  
     ```bat
     flutter build apk --release
     ```  
   锁被释放后，这次命令就有机会抢到锁并完成打包。

## 二、Pub 镜像 TLS 报错（依赖拉取失败）

### 现象
锁释放后，构建会先执行 `flutter pub get`，若出现类似：
```text
Got TLS error trying to find package xxx at https://pub.flutter-io.cn.
pub get failed (server unavailable) -- attempting retry 1 in 1 second...
```
说明是**拉取依赖**失败，不是 Flutter 没装好。

### 原因
- 当前环境把 **PUB_HOSTED_URL** 设成了国内镜像（如 `https://pub.flutter-io.cn`）。  
- 该镜像有时会出现 TLS/证书问题，导致 `pub get` 失败，进而整个 `flutter build apk` 失败，所以看起来像“不能打包”。

### 解决办法

在**执行打包的那次终端**里，先切到官方源再打包：

**PowerShell：**
```powershell
$env:PUB_HOSTED_URL='https://pub.dev'
$env:FLUTTER_STORAGE_BASE_URL='https://storage.googleapis.com'
cd e:\steamGame
flutter build apk --release
```

**cmd：**
```bat
set PUB_HOSTED_URL=https://pub.dev
set FLUTTER_STORAGE_BASE_URL=https://storage.googleapis.com
cd /d e:\steamGame
flutter build apk --release
```

项目里的 **`build_apk_full.bat`** 里已经写了这两行环境变量，用脚本打包时会自动用官方源。

## 发布到 Google Play：Google 登录配置（可选）

应用在「加入愿望单」时会要求用户用 Google 账号登录。若要在真机或上架后正常使用登录功能，需要在 [Google Cloud Console](https://console.cloud.google.com/) 配置 OAuth：

1. 新建或选择项目 → **API 和服务** → **凭据** → **创建凭据** → **OAuth 2.0 客户端 ID**。
2. 应用类型选 **Android**，包名填 `com.steamdeal.alert`（与 `android/app/build.gradle` 里 `applicationId` 一致）。
3. 获取 SHA-1：在项目根目录执行 `cd android && ./gradlew signingReport`（Windows 用 `gradlew.bat signingReport`），在输出里复制 **release** 的 SHA-1 填入控制台。
4. 创建后得到客户端 ID，无需在应用里写死；Google Sign-In 插件会使用默认配置。

未配置时，在模拟器或未签名的 debug 包上登录可能失败，属正常；配置好 release 包名与 SHA-1 后，正式包即可登录。

## 版本号与 Google Play 发布

- **版本号位置**：`pubspec.yaml` 里 `version: 1.0.1+2`。格式为 `版本名+版本号`（如 `1.0.1+2`）。
- **每次上架新包**：在 Google Play 控制台提交的 APK/AAB 的 **versionCode**（即 `+` 后面的数字）必须大于上一版。建议每次发布前把 `version` 改为例如 `1.0.2+3`（名可自定，号必递增）。
- **应用内更新**：应用已接入 Google Play 应用内更新。用户从 Play 商店安装后，启动约 2 秒会检查是否有新版本；若有会弹窗「发现新版本，是否立即更新？」。**仅当应用从 Google Play 安装时有效**，本地安装的 APK 不会走 Play 更新。

## Google Play 上传用包说明

- **本次已生成的包**：  
  - **APK**（可直接用于测试或上传）：`build\app\outputs\apk\release\app-release.apk`（约 18.6MB）。  
  - 应用名称已设为 **Steam Deal Alert**（在 AndroidManifest 的 `android:label`）。

- **推荐用 AAB 上传**（体积更小、Play 自动优化）：在 **Cursor 外** 的 cmd 中执行（避免抢锁）：  
  ```bat
  set PUB_HOSTED_URL=https://pub.dev
  set FLUTTER_STORAGE_BASE_URL=https://storage.googleapis.com
  set ANDROID_HOME=D:\Android\Sdk
  set JAVA_HOME=E:\java\jdk-11.0.9
  cd /d e:\steamGame
  flutter pub get
  flutter build appbundle --release
  ```  
  生成的 AAB 在：`build\app\outputs\bundle\release\app-release.aab`，在 Play 控制台「发布」→「应用版本」中上传此文件。

- **上传前建议**：  
  1. **包名**：已改为 `com.steamdeal.alert`（符合 Google Play 要求，不再使用 `com.example`）。若需改用自有域名，可同步改 `android/app/build.gradle` 的 `applicationId` 和 `AndroidManifest.xml` 的 `package` 及 Kotlin 包目录。  
  2. **签名（必做）**：若未配置 release 签名，打出来的 AAB 是 **debug 签名**，Play 会拒绝并提示“以调试模式签名”。  
     - 已配置按 `android/key.properties` 做 release 签名。  
     - 首次：在 **android** 目录双击 **create_keystore.bat** 生成 **upload-keystore.jks**，牢记密码。  
     - 将 **key.properties.example** 复制为 **key.properties**，填写 storePassword、keyPassword、keyAlias=upload、storeFile=../upload-keystore.jks。  
     - 再执行 `flutter build appbundle --release` 后上传新 AAB 即可发布。

## 方便签名的方法

- **一键打出已签名 AAB**：在项目根目录 **e:\\steamGame** 下双击运行 **`打包发布_带签名.bat`**。  
  - 脚本会自动设好 ANDROID_HOME、JAVA_HOME、官方 pub 源，并检查是否已有 `android/key.properties` 和 `android/upload-keystore.jks`。  
  - 若已配置好，会直接执行 `flutter pub get` 和 `flutter build appbundle --release`，生成的 **build\\app\\outputs\\bundle\\release\\app-release.aab** 即为带 release 签名的包，可直接上传 Play。  
  - 若未配置，会提示先运行 `android\create_keystore.bat` 并填写 `key.properties`，再重新运行本脚本即可。

- **用 Android Studio 签名**：用 Android Studio 打开项目下的 **android** 目录，菜单 **Build → Generate Signed Bundle / APK**，选择 **Android App Bundle**，选择已有或新建 keystore，按向导完成即可。适合习惯图形界面的情况。

## 总结

| 现象 | 原因 | 做法 |
|------|------|------|
| 一直卡在 “Waiting for... startup lock” | Cursor/IDE 的 dart 进程占着 Flutter 锁 | 在 Cursor 外 cmd 打包，或先结束 dart 再打包 |
| pub get 报 TLS / server unavailable | 国内镜像不稳定 | 用官方源（或直接运行 `build_apk_full.bat`） |

你的 Flutter 环境本身没问题，按上面处理“锁”和“镜像”后，就可以正常打出 APK。
